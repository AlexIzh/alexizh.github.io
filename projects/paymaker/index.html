PayMaker 2

Apr, 2017
A macOS application which helps to manage and create your payments according to ISO 20022.

Role: macOS Developer

Team: I’m only one developer on this project.

Responsibility: Developing the whole application.

Technologies: CoreData, Swift, SQLite, Sparkle, Storyboards, Apple script

Description

The main purpose of the app is generating XML file according to ISO 20022 from payments, so a user can send this file to his bank. For that, I implemented ISO 20022 serialization along with DTA serialization since it’s still used by some banks in Switzerland.

For the project, I implemented this parser which allow combining queries by logical operands (kind of predicate). The app uses NSPersistenceDocument as default user’s document type (SQLite database which managed by CoreData) but at the same time, the app can work with a pure SQLite database for importing data from other platforms/products. For example, it uses SQLite for importing data from the window’s version. Also, data could be imported from CSV file, from the old version of PayMaker (it had another structure of the document and another SQLite DB structure) or from other applications. For that, I implemented a CSV parser with supporting different delimiters, quotes and other things necessary for CSV. But it’s another talk.

Also, since it’s an app about payments, I had to implement IBAN checking. For that, I implemented IBAN validator and formatter according to this document. The validator uses BBAN format for validating.

One more interesting feature is smart folders. It works and looks exactly like smart folders inside Mail.app.

The application supports Apple scripting for exporting payments to user’s formats, like Numbers, iCash and so on. A user can write custom scripts which he needs and export data to any format he needs.

Of course, there a lot of interesting things yet which I missed here, if you are interested, feel free to write an email to me.
