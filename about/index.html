About

Hi!
My name is Alex, I’m an iOS and macOS developer with a quite big experience.

My favorite book (about programming) is Working effectively with legacy code. I like it because it contains very interesting things how you could work with a bad structured/untested/old code, and as we all know, it is most code. But at the same time, all we (or almost all) want to provide good services to our customers and make them happy (or at least a bit happier).

How I became a developer?

Here is my journey from a moment when I started to learn programming to these days:

I started to learn programming at school when I was around 13-14 years old.
It was Pascal programming language. I liked it a lot, so I started to look at other more popular languages.
Then I started to learn C. Can’t say it was a successful try, it was quite difficult for me in those days.
Later I started to look for any easier language to learn and a lot of articles on the internet said that I should try to learn Python. And of course, I started to learn it. It was more successful try than with C, but still, I didn’t write any complete application (even small) with that just because I didn’t find how to write UI with Python. After that, I came back to C and started to learn C++ with using C++ Builder for Windows. It’s the first time I implemented a small real application with UI.
That’s how my way was started.

After that, I implemented/tried a lot of languages and platforms. I even tried and learned Perl a bit and wrote a small chat-bot on it! But don’t ask me to repeat it, it was years ago and since then I even didn’t see Perl anymore.
Some other examples what I did for fun/learning purposes (what is not related to my job/iOS/macOS development):

  • C/GTK+ on Linux - a few small UI applications, tic-tac-toe, sea battle
  • A small unfinished game on C# for Bada(old Samsung mobile OS) (by the way, I tried C# with Unity last couple of years)
  • Parsing a math expression from a string and draw a graph of the function on Delphi
  • Simulation of people evacuation in case of fire - Java (very interesting project)

And in 2011 one outsourcing company hired me as a junior and I started to learn iOS development. Since that I also learned and tried a lot, a lot of projects passed away, a lot of time was spent on development and after all this time, I still love it and like to learn new things and try to develop something even in my free time (of course, when I have it after my family).
