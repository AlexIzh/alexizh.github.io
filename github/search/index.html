SearchQueryParser

GitHub Repository
Jul, 2017
MacOS, iOS, Swift
Parsing a user's search query to a predicate or filtering function with supporting binary operators.

The parser recognizes brackets, quotes, binary and unary operators. It works like parsing of a string to calculate value inside calculators or displaying graphs. First, we tokenize every part of a string, then Reverse Polish Notation is generated from the parts and after that, we can calculate the result or generate any predicate/filtering function which we need. It's very customizable and could support any custom unary and binary operators, brackets, and quotes.

By default it supports:
" - quotes (inside quotes the phrase won't be parsed and will be used like one item)
() - brackets, it's used exactly like inside NSPredicate or any calculator
NOT or ! - boolean NOT operator. Exclude the value from a search result
AND or & or space - boolean AND. The character represents a boolean operator when there are terms to both sides of the character. In this case, represents a Boolean AND by default, or a Boolean OR if specified by .spaceMeansOR option.
OR or | - boolean inclusive OR

Example:

let builder = DefaultPredicateBuilder() { NSPredicate(format: "(name LIKE[c] %@) OR (job LIKE[c] %@)", $0, $0) }
self.predicate = builder.build(from: searchField.stringValue)
